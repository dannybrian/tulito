• Fully locked-down control over browser scroll and zoom
  • Automatic touch events with Hammer.js
  • Native-like delayed button behaviors
  • Draggable, swipe-able, shove-able content panes in any direction
  • Smooth pane animations
  • Hidden panes in any direction
  • Back menu panes to the left and right
  • Scrollable elements with momentum using iScroll (optional)
  • Responsive scaling design between devices and orientations
  • Compatible with any JavaScript or CSS library or framework
  • Everything configured through simple HTML data attributes
  • PhoneGap-ready

Modern Web technologies — HTML5, CSS3, vector icon sets, Web fonts, JavaScript libraries — make it easier to build great mobile apps, but putting the pieces together to get the layout, animation, and touch behaviors just right can still be a challenge. Tulito aims to give development teams a starting point for those behaviors, and a fastidiously simple design as a prototype from which developers can learn.

Tulito is a free template and library by Danny Brian for building mobile apps with HTML5, CSS3, and JavaScript. It captures critical behaviors for creating a great mobile user experience, and gives you a starting point for your own clean design. Tulito uses the excellent Hammer.js, LESS, iScroll, Source Sans Pro, and Font Awesome projects to enhance functionality. (Of these, only Hammer.js is required.) It is compatible with any JavaScript or CSS library. It can be deployed via Web or PhoneGap.

Visit tulito.org to download and improve the template.
